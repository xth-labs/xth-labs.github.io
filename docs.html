Quick Start

XThereon is designed to be used without complex setup. The fastest way to participate is by running the CPU miner, which automatically manages the node and wallet.

Recommended (Windows)

  1. Download the XTH CPU Miner from the Download page
  2. Run the installer and launch the miner
  3. The node will sync automatically
  4. Mining begins once the network is ready

Linux (Ubuntu)

  1. Extract the miner archive
  2. Run ./xthminer
  3. Data is stored in ~/.xthereon

XTH is not pre-allocated or sold. Coins enter circulation only through mining.

Mining XTH

Mining is the primary way XTH enters circulation. By mining, you help secure the network and earn XTH in return.

CPU Mining

The official XTH miner uses CPU-based proof-of-work. No pools, accounts, or third-party services are required.

  • ✔ Automatic wallet creation
  • ✔ Automatic node startup
  • ✔ Solo mining by default

Important Notes

  • Mining uses CPU resources
  • Closing the miner stops mining
  • Block rewards mature after confirmation

Running a Full Node

Running a full node gives you maximum control and helps decentralize the XThereon network.

A full node:

  • Validates every block and transaction
  • Enforces consensus rules independently
  • Does not rely on third parties

Who should run a node?

Anyone can run a node, but it is especially recommended for:

  • Miners
  • Infrastructure operators
  • Long-term participants

Dedicated server binaries will be published once the network matures.

RPC Commands

XThereon nodes expose a JSON-RPC interface for automation, scripting, and integration with external services.

Common Commands

xthereon-cli getblockcount
xthereon-cli getwalletinfo
xthereon-cli getnewaddress
xthereon-cli getbalance

RPC access is local by default and should not be exposed to the public internet.

A full RPC reference will be expanded as the project evolves.
